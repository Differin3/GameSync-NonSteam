# Инструкция по обновлению плагина GameSync NonSteam

## Проблема
Плагин GameSync NonSteam несовместим с текущей версией SteamUI после недавнего обновления Steam. Фронтенд-бандл плагина содержит устаревший код, который не может быть загружен в новую архитектуру Steam.

## Решение
Были созданы обновленные версии файлов компонентов для совместимости с новой архитектурой SteamUI.

## Шаги по обновлению

### 1. Обновление зависимостей
Выполните в терминале:
```bash
cd "c:\project\GameSync NonSteam"
npm install
```

### 2. Замена файлов
Запустите скрипт обновления:
```bash
python update_plugin.py
```

Или вручную замените следующие файлы:
- `src/index.tsx` ← `src/index_new.tsx`
- `src/components/GameSyncMain.tsx` ← `src/components/GameSyncMain_new.tsx`
- `src/components/GameList.tsx` ← `src/components/GameList_new.tsx`
- `src/components/Settings.tsx` ← `src/components/Settings_new.tsx`
- `src/components/GamePathEditor.tsx` ← `src/components/GamePathEditor_new.tsx`

### 3. Сборка плагина
Выполните в терминале:
```bash
npm run build
```

### 4. Установка плагина
1. Создайте архив с файлами плагина:
   - Создайте папку `dist-plugin`
   - Скопируйте: `dist/`, `backend/`, `plugin.json`, `requirements.txt`
   - Заархивируйте в `gamesync-nonsteam.zip`

2. Установите плагин на Steam Deck:
   - Скопируйте `gamesync-nonsteam.zip` на Steam Deck
   - Откройте Decky Loader → Developer → Install from ZIP
   - Выберите архив и установите

## Основные изменения в коде

1. Обновлены импорты для использования актуального API Decky
2. Компоненты адаптированы под новую архитектуру PanelSection
3. Исправлены проблемы с JSX синтаксисом
4. Обновлен формат сборки с CommonJS на IIFE

После выполнения этих шагов плагин должен корректно работать с новой версией SteamUI.
